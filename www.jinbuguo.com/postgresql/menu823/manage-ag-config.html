19.4. 数据库配置

回顾一下章17我们知道 PostgreSQL 服务器提供了大量的运行时配置变量。你可以为许多这样的变量设置特定于数据库的缺省数值。

比如，如果由于某种原因，你想关闭某个数据库上的 GEQO 优化器，你就不得不要么在一开始就在所有数据库中关闭它，要么是保证每个连接过来的客户端都很小心地发出了 SET geqo TO off; 命令。要令这个设置在特定数据库里成为缺省，你可以执行下面的命令

ALTER DATABASE mydb SET geqo TO off;

这样将保存该设置(但不是立即设置它)。在随后的连接中它将表现的像在会话开始后马上调用 SET geqo TO off; 一样。请注意用户仍然可以在该会话中更改这个设置(它只是缺省)。要撤消这样的设置，使用 ALTER DATABASE dbname RESET varname;
